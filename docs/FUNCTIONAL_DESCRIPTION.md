# ПРИЗМА — Функциональное описание

**Версия:** 1.0
**Дата:** 29.03.2026
**Статус:** Продуктивная эксплуатация (НТЦ-2Ц)

---

## 1. Общие сведения

### 1.1 Назначение системы

ПРИЗМА (**П**ланирование **Р**есурсов **И** **З**адач, **М**ониторинг и **А**нализ) — веб-система управления работами, планирования и контроля исполнения задач в научно-техническом центре (НТЦ). Система обеспечивает полный цикл управления: от формирования производственного плана до контроля исполнения и аналитики.

### 1.2 Технологический стек

| Компонент | Технология |
|---|---|
| Серверная часть | Python 3.11, Django 3.2 |
| База данных | PostgreSQL 15 |
| Веб-сервер | Gunicorn |
| Контейнеризация | Docker |
| Клиентская часть | Vanilla JavaScript (SPA-модули) |
| Диаграмма Ганта | dhtmlxGantt (GPL) |

### 1.3 Архитектурные принципы

- Все модули — одностраничные приложения (SPA) внутри Django-шаблонов
- REST API для всех операций с данными
- Ролевая модель доступа на уровне API
- Аудит всех действий пользователей
- Развёртывание на внутренних серверах предприятия (данные не покидают контур)

## 2. Модули системы

### 2.1 Стартовая страница (Дашборд)

**URL:** `/accounts/dashboard/`

**Назначение:** Персональная сводка для текущего пользователя.

**Функции:**
- Отображение персонального плана задач (текущие, просроченные, выполненные)
- Для руководителей — сводка по подразделению (загрузка, статусы)
- Быстрый доступ к основным модулям
- Виджет уведомлений (дедлайны, изменения)

---

### 2.2 Управление проектами (УП)

**URL:** `/works/projects/`
**Доступ:** Администратор

**Назначение:** Ведение реестра проектов и изделий.

**Функции:**

| Функция | Описание |
|---|---|
| Реестр проектов | Карточки проектов с полным и кратким наименованием, шифром |
| Изделия проекта | Справочник изделий (продуктов) в рамках каждого проекта |
| Метрики проекта | Количество задач ПП, количество изделий |
| CRUD | Создание, редактирование, удаление проектов и изделий |

**Поля проекта:**
- Полное наименование (`name_full`)
- Краткое наименование (`name_short`)
- Шифр (`code`)

**Поля изделия:**
- Наименование (`name`)
- Краткое наименование (`name_short`)
- Шифр (`code`)

---

### 2.3 Производственный план (ПП)

**URL:** `/works/production-plan/`
**Доступ:** Чтение — все; редактирование — роли с правом записи

**Назначение:** Формирование и ведение производственного плана подразделения по проектам/изделиям.

**Функции:**

| Функция | Описание |
|---|---|
| Проекты ПП | Список планов (по проектам), создание нового плана |
| Табличный редактор | Inline-редактирование ячеек (двойной клик) |
| Автогенерация кода строки | Формат: `{Краткое_имя_проекта}.{N}`, атомарный счётчик |
| Фильтрация | Мульти-фильтры по каждому столбцу, фильтр по статусу |
| Сортировка | По любому столбцу (по возрастанию/убыванию) |
| Поиск | Полнотекстовый поиск по всем столбцам |
| Зависимости задач | FS/SS/FF/SF с лагом в днях, обнаружение циклов (BFS) |
| Выравнивание дат | Каскадное выравнивание по зависимостям |
| Синхронизация ПП→СП | Перенос строк ПП в сводное планирование |
| Статусная панель | Счётчики: Всего / Выполнено / Просрочено / В работе |
| Адаптивная вёрстка | Карточный вид на мобильных устройствах |

**Поля строки ПП:**
- Код строки, номер заказа, этап, веха, номер работы
- Обозначение документа, наименование работы
- Дата окончания, листов А4, норма, коэффициент, трудоёмкость
- Центр, отдел, начальник сектора, исполнитель, тип работы

### 2.4 Сводное планирование (СП) — План/отчёт

**URL:** `/works/plan/`
**Доступ:** Чтение — все; редактирование — роли с правом записи

**Назначение:** Единый план-отчёт по всем задачам подразделения с помесячным распределением трудозатрат.

**Функции:**

| Функция | Описание |
|---|---|
| Таблица задач | С помесячными колонками плановых часов |
| Модальное редактирование | Полная карточка задачи в модальном окне |
| Отчёты по задаче | Фиксация выполнения, прикрепление к задаче |
| Мульти-фильтры | По каждому столбцу: отдел, сектор, исполнитель, тип и т.д. |
| Статусная фильтрация | Все / Выполненные / Просроченные / В работе |
| Ошибки планирования | Кнопка проверки: пересечения с отпусками, отсутствие часов и др. |
| Зависимости задач | Просмотр и управление связями между задачами |
| Плотность строк | Переключение: компактный / средний / просторный |
| Управление столбцами | Видимость, порядок, сохранение ширины на сервере |
| Бесконечная прокрутка | Ленивая загрузка записей порциями |
| Список исполнителей | JSONField со списком соисполнителей |
| Помесячные часы | JSONField `plan_hours` с разбивкой по месяцам |

**Блокировка полей:** Задачи, перенесённые из ПП (`from_pp`), имеют заблокированные поля: наименование, номер, обозначение, этап, обоснование.

### 2.5 Журнал извещений (ЖИ)

**URL:** `/works/notices/`
**Доступ:** Чтение — все; редактирование — администратор

**Назначение:** Учёт извещений об изменениях (ИИ/ПИ) с контролем сроков погашения.

**Функции:**

| Функция | Описание |
|---|---|
| Реестр извещений | Таблица с фильтрацией, сортировкой, поиском |
| Два режима | Авто (из отчёта по задаче) и ручной (все поля вручную) |
| Статусы | active (ИИ), expired (ПИ+истёк), closed_no, closed_yes |
| Погашение | Номер погашающего извещения, дата, исполнитель |
| Проверка дубликатов | Серверная проверка уникальности номера |
| Мульти-фильтры | По каждому столбцу |
| Бесконечная прокрутка | Ленивая загрузка |

**Автоматическое создание:** При установке `task_type = "Корректировка документа"` в отчёте система автоматически создаёт запись в журнале.

### 2.6 Аналитика

**URL:** `/works/analytics/`
**Доступ:** Зависит от роли (видимость данных по подразделению)

**Назначение:** Визуальная аналитика по загрузке, исполнению и срокам.

**Три аналитических доски:**

#### 2.6.1 Доска руководителя (загрузка)

- Загрузка по отделам (помесячно)
- Предупреждения о дедлайнах (просроченные / горящие)
- Распределение статусов задач по подразделениям
- Фильтры: год, отдел

#### 2.6.2 Доска сотрудника

- Персональная загрузка по месяцам
- Список задач (статус, дедлайн, часы)
- Итоговые карточки: всего часов, в работе, выполнено, просрочено
- Фильтры: год, сотрудник

#### 2.6.3 Доска ПП

- Метрики производственного плана: % выполнения, статусы
- Прогресс по проекту / этапу
- Разбивка по трудоёмкости

## 4. Ролевая модель

### 4.1 Роли

| Роль | Код | Запись | Область видимости |
|---|---|---|---|
| Администратор | `admin` | Да | Все данные, управление пользователями |
| Руководитель НТЦ | `ntc_head` | Да | Все данные НТЦ |
| Зам. руководителя НТЦ | `ntc_deputy` | Да | Все данные НТЦ |
| Начальник отдела | `dept_head` | Да | Данные своего отдела |
| Зам. начальника отдела | `dept_deputy` | Да | Данные своего отдела |
| Начальник сектора | `sector_head` | Да | Данные своего сектора |
| Исполнитель | `user` | Нет | Только назначенные задачи |

### 4.2 Делегирование полномочий

Механизм временной передачи прав (`RoleDelegation`):
- Делегирующий передаёт полномочия замещающему на срок (`valid_until`)
- Флаг `can_write` — с правом записи или только чтение
- Проверяется при каждой операции записи

### 4.3 Правила доступа к данным

- **Производственный план** — общий документ, видимый всем
- **Сводное планирование** — фильтрация по подразделению пользователя
- **Аналитика** — данные в пределах области видимости роли
- **Справочники** — редактирование только администратором

## 5. Сквозные функции

### 5.1 Командная палитра

**Вызов:** `Ctrl+K` (Windows) / `Cmd+K` (macOS)

Быстрый поиск и навигация по всем модулям системы:
- Нечёткий поиск по названиям страниц и ключевым словам
- Контекстные действия: создание задачи, экспорт, смена темы
- Навигация стрелками, выбор по Enter

### 5.2 Уведомления

- Автоматическая генерация уведомлений о приближающихся дедлайнах
- Отметка «прочитано» / «прочитать все»
- Счётчик непрочитанных в интерфейсе

### 5.3 Комментарии

- Комментарии к задачам (`/api/comments/`)
- Создание и удаление

### 5.4 Песочница изменений (Changesets)

Механизм групповых изменений с согласованием:
- Создание набора изменений (changeset)
- Добавление элементов (changeset items)
- Отправка на согласование → одобрение / отклонение
- Просмотр разницы (diff)
- Повторное открытие

### 5.5 Интерактивное обучение

17-шаговый обучающий тур по всем модулям:
- Подсветка элементов интерфейса (spotlight)
- Информационные карточки с описанием
- Автонавигация между страницами
- Сохранение прогресса (localStorage + сервер)
- Перезапуск из профиля

### 5.6 Темы оформления

Четыре темы: Светлая, Сумеречная, Тёмная, Системная (по настройке ОС).

## 6. Безопасность

| Механизм | Описание |
|---|---|
| Аутентификация | Сессионная (Django), Redis-хранилище (production) |
| CSRF-защита | Токены на всех формах и API-запросах |
| Rate-limiting | Логин: 10/60с, Регистрация: 5/300с, API: 300/60с |
| Ролевая модель | Проверка прав на уровне каждого API-эндпоинта |
| Аудит | Журналирование всех действий пользователей |
| Развёртывание | Внутренние серверы, данные не покидают контур предприятия |

## 7. API

Полный REST API (72+ эндпоинта) для всех операций. Основные группы:

| Группа | Префикс | Назначение |
|---|---|---|
| Задачи | `/api/tasks/` | CRUD задач, массовые операции |
| Производственный план | `/api/production_plan/` | CRUD строк ПП, синхронизация |
| Проекты ПП | `/api/pp_projects/` | CRUD проектов ПП |
| Проекты УП | `/api/projects/` | CRUD проектов, изделия, метрики |
| Зависимости | `/api/dependencies/` | CRUD зависимостей, выравнивание |
| Отчёты | `/api/reports/` | CRUD отчётов по задачам |
| Журнал | `/api/journal/` | CRUD извещений |
| Аналитика | `/api/analytics/` | Загрузка, метрики, дашборды |
| Календарь | `/api/work_calendar/` | Рабочие часы, праздники |
| Справочники | `/api/directories/` | Универсальный справочник |
| Пользователи | `/api/users/` | Управление пользователями |
| Делегирование | `/api/delegations/` | Временная передача прав |
| Настройки | `/api/col_settings/` | Настройки столбцов и UI |
| Уведомления | `/api/notifications/` | Системные уведомления |
| Комментарии | `/api/comments/` | Комментарии к задачам |
| Отпуска | `/api/vacations/` | Учёт отпусков |
| Командировки | `/api/business_trips/` | Учёт командировок |
| Обратная связь | `/api/feedback/` | Замечания и предложения |
| Песочница | `/api/changesets/` | Групповые изменения |
| Аудит | `/api/audit_log/` | Журнал действий |
| Здоровье | `/api/health/` | Проверка доступности |

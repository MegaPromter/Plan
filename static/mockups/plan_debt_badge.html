Бейджи «план / долг» — варианты для шапки СП

Сейчас один бейдж «🕓 3 503,8 ч» суммирует часы плановых задач + долги месяца. Ниже — варианты визуально разделить две величины. Данные в примерах: план = 2 150 ч, долг = 1 354 ч, всего = 3 504 ч. Норма = 2 024 ч (12 сотр × ≈168 ч). Загрузка по плану = 106%, с долгом = 174%.

Одна капсула, две цветные зоны. Клик по зоне → фильтр таблицы (бакет overdue/unplanned).

Что нужно поменять на сервере

В MonthSnapshotView сейчас отдаётся общая сумма часов. Чтобы нарисовать сплит-бейдж, нужно дополнительно выдать два поля:

  • planned_hours — часы задач, у которых plan_hours_all[monthKey] > 0 (включая перегруз по плану)
  • debt_hours — часы задач-долгов (просрочены, отчёта нет, на текущий месяц часы не проставлены). Тут вопрос: считать по labor или по plan_hours_all прошлого периода — нужно решить.
